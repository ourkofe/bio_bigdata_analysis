# bio_bigdata_analysis

생물정보학(바이오빅데이터) 분석 파이프라인을 직접 실습하며 경험을 정리하는 저장소입니다.
분석 종류별로 디렉토리를 분리해서, 각 분석의 파이프라인 이해와 실습 과정을 기록합니다.

> 원본 시퀀싱 데이터(FASTQ/BAM/참조 유전체 등)는 용량이 커서 이 저장소에 올리지 않습니다.
> 각 분석 디렉토리의 README에 데이터 출처와 다운로드 명령어를 기록해 재현 가능하게 합니다.

## 분석 목록

| 디렉토리 | 분석명 | 상태 |
|---|---|---|
| [`wes_germline_variant_calling/`](wes_germline_variant_calling/) | WES Germline SNV/Indel Calling | 🚧 진행 중 |
| `wgs_.../` | (예정) WGS 분석 | ⏳ 예정 |
| `rna_seq_.../` | (예정) RNA-seq 분석 | ⏳ 예정 |

## 디렉토리 구성 원칙

각 분석 디렉토리는 아래 구조를 따릅니다:

```
<분석명>/
├── README.md          # 분석 목적, 배경 개념, 파이프라인, 진행 상황
├── environment.yml      # conda 환경 (분석마다 필요한 툴이 다르므로 개별 관리)
├── scripts/              # 단계별 실행 스크립트 (실습하면서 하나씩 추가)
├── notes/
│   └── log.md             # 날짜별 실습 일지
├── data/                  # (gitignore) 원본/중간 데이터, 로컬에만 존재
└── results/               # 최종 결과물, QC 리포트 등 (용량 작은 것만 커밋)
```

## 공통 규칙

- 대용량 파일(FASTQ, BAM, 참조 FASTA 등)은 절대 커밋하지 않음 → 루트 `.gitignore`에서 공통 관리
- 각 분석은 독립적으로 진행 가능하도록 디렉토리 단위로 완결성을 유지
- 실습 과정(에러, 시행착오 포함)은 각 분석의 `notes/log.md`에 기록
